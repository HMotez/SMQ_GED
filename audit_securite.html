Plan de correction — Priorités
1
CRITIQUE   Protection Brute Force — Rate Limiting
Installer express-rate-limit sur le endpoint POST /api/auth/login. Bloquer le compte 15min après 3 tentatives échouées. Fichier : authController.js:116 + authRoutes.js:11
2
CRITIQUE   Invalidation du JWT à la déconnexion
Implémenter une blacklist de tokens en Redis ou en base de données. La fonction logout() doit ajouter le JTI du token à cette liste. Fichier : authController.js:301
3
CRITIQUE   HTTPS / SSL en production
Configurer un certificat SSL (Let's Encrypt/Certbot) dans Nginx. Rediriger le port 80 vers 443. Fichier : nginx.conf:11, docker-compose.yml
4
IMPORTANT   Headers de sécurité HTTP (helmet.js)
Installer npm install helmet et ajouter app.use(helmet()) dans server.js. Ajoute automatiquement CSP, X-XSS-Protection, HSTS, X-Frame-Options, X-Content-Type-Options.
5
IMPORTANT   Restreindre la politique CORS
Remplacer app.use(cors()) par app.use(cors({ origin: process.env.ALLOWED_ORIGINS })) avec la liste des domaines autorisés. Fichier : server.js:20
6
IMPORTANT   Renforcer la politique de mots de passe
Exiger minimum 8 caractères avec au moins une majuscule, un chiffre et un caractère spécial. Fichier : authController.js:249 et :401
7
MOYEN   Masquer les messages d'erreur internes
Remplacer res.json({ error: err.message }) par un message générique "Erreur serveur." avec log interne uniquement. Fichier : server.js:190
8
MOYEN   Ne pas exposer le port backend 4000
Supprimer la section ports: "4000:4000" du service backend dans docker-compose.yml:88. L'API doit être accessible uniquement via Nginx (port 80/443).
